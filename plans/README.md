## Direction findings (not ranked against bugs)

1. **Make catalog browsing public or make the CTA honest.** The product pitch says customers can discover packages, but the landing page routes “Browse packages” to `/signup` (`frontend/src/app/page.tsx:45-47`) and the real catalog is under the authenticated dashboard group. A public read-only catalog would improve the reviewer/user funnel; the trade-off is exposing pricing/catalog APIs without auth, which already appears intended for `MapCatalogEndpoints`.

2. **Add a reviewer-grade E2E smoke path.** The README’s 60-second tour (`README.md:35-46`) is exactly the right Playwright script: signup → catalog → provision → resource detail → billing/admin health. This would be high demo value, but it depends on stable seed/provisioning and should not precede core API correctness fixes.

3. **Turn the documented “what next” observability/SSE ideas into a spike, not a full build.** `DESIGN.md:81-92` names LISTEN/NOTIFY, OpenTelemetry, Prometheus, and webhooks. The architecture makes these plausible, but the current demo will benefit more from correctness/security/test gates first; a spike should define event contracts and operational metrics before implementation.
