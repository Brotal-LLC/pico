# Pico — OpenSpec Entry Point

This project uses OpenSpec for spec-driven development. See the
`~/.hermes/skills/spec-driven-dev/SKILL.md` skill for the full workflow.

## Commands (type as prompts to the AI assistant)

- `/opsx:propose <name> <why>` — create a change proposal
- `/opsx:apply` — implement tasks top-to-bottom
- `/opsx:archive` — fold completed change into specs
- `/opsx:sync` — check for drift between specs and code

## Current specs

See `openspec/specs/` for the current system specification.
See `openspec/changes/` for active proposals.
